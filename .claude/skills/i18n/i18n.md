# Skill: i18n for Scandinordic Pro

## Languages
en (English), fi (Finnish), sv (Swedish), da (Danish)

## File location
`lib/i18n.ts`

## Rules
1. Every new UI string MUST be added to ALL 4 language blocks
2. Never add duplicate keys — check first with grep
3. Keys use camelCase: `addEmployee`, `grossSalary`, `netPay`
4. Use formal business language for fi/sv/da (not casual)

## Usage in components
```ts
import { useTranslation } from '@/lib/i18n';
const { t } = useTranslation();
// then:
<Text>{t('addEmployee')}</Text>
```

## Adding a new key — checklist
- [ ] Key doesn't exist yet in i18n.ts
- [ ] Added to `en` block
- [ ] Added to `fi` block  
- [ ] Added to `sv` block
- [ ] Added to `da` block
- [ ] No duplicate key errors after save
